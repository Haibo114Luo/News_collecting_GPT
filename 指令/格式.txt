# 格式.txt
【分类标签】
政策支持｜政策收紧｜项目开工/许可｜并网/投运｜招标/中标｜投资/融资｜并购重组｜供应链扩产/本地化｜供应链瓶颈/扰动｜成本负面冲击（含关税/原材料/电价/物流）｜成本缓解（含价格回落）｜技术突破/试点｜市场/统计数据｜价格/指数变动｜电网接入/排队/扩容｜电力采购/PPA/电价机制｜算力与设备供应链（芯片/服务器/液冷）｜能效/碳/用水与合规｜园区选址/土地/数据基础设施许可｜运营中断/安全事件（停电/火灾/网络）

【输出格式（必须全部用 ```markdown 代码块包裹）】
- 代码块中采用md语言；仅纯文本排版，不含任何UI控件/按钮/组件/navlist。
- 组内按重要性降序（对行业边际影响更大者在前）。
- 每条固定字段：
  - 日期：报道YYYY-MM-DD / 事件YYYY-MM-DD
  - 地区：国家/地区（可加省州/海域）
  - 类别：从标签中选
  - 事件：50–100字关键事实（含容量/金额/机制要点/里程碑）
  - 影响：50–100字，给出面向行业的边际影响判断：明确该事件相对既有共识/规则新增了什么信息，以及因此对【所选的类别】具体产生的方向性变化；禁止空话。
  - 来源：URL: https://... （原始URL纯文本，不用[标题](链接)，不能仅写出主站，必须具体到描述新闻的网页本身）

示例：
# 2025.11.18
## 中国
#### 1.
- **日期**：报道YYYY-MM-DD / 事件YYYY-MM-DD
- **地区**：中国（如含具体省份/城市则加括号）
- **类别**：政策支持 / 项目开工 / 并购重组 / …
- **事件**：（MM月DD日《媒体全称》报道）MM月DD日，……（50–100字行业关键事实）
- **影响**：50–100字，给出面向行业的边际影响判断
- **来源**：https://original.url/xxxx/xxx
#### 2.
- …
## 美国
#### 1.
- …
## 欧洲
### 英国
#### 1.
- …
其中，
来源：新闻所对应的网址纯文本（http/https开头的原始URL）。**必须指向具体的新闻而非仅提供新闻网站的地址**；必须指向具体显示此条新闻的网页，而非仅仅是形如https://www.reuters.com的不定向域名。
禁止内嵌超链接、引用块、脚注或UI控件或任何括号包装；不要使用`[标题](链接)`格式。如果被系统要求必须引用，则放在代码块之外。
不同条目之间需以额外空行分隔，以便 Markdown 引擎正确识别结构层级。代码块内部仅使用 Markdown 文本排版符号（如标题、列表、粗体标注），不包含任何 UI 控件、可交互组件、HTML/JS 构件或 navlist。所有输出均为纯文本，不包含执行逻辑；

【Markdown层级】
H1（#）: 顶部使用 # 写本次检索任务标题。例如：# 2025.11.18
H2（##）: 不含有H3层级：中国大陆；美国。含有H3层级：欧洲；其余亚洲；中东与非洲；其他。例如：## 中国
H3（###）:欧洲（英国、德国、法国、荷兰、丹麦/北欧）；其余亚洲（印度、日本/韩国、东南亚）；中东与非洲（中东与北非、撒哈拉以南非洲）；其他（拉丁美洲、大洋洲、全球/多地区）。例如：### 欧洲—英国
H4（####）: 具体项目编号，例如：#### 1.
